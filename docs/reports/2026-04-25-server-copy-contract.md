# 2026-04-25 服务端文案契约与多语言准备报告

## 1. 现有问题

上一轮已经完成 INR 校正值、检测方式、检测周期和首页提醒的产品完善，但仍存在一个实现层面的长期风险：客户端页面里散落了大量业务中文文案。后续如果要支持多语言、文案 A/B、医生版/家庭版差异化或监管口径调整，Flutter、小程序、静态原型会分别修改，容易出现不一致。

## 2. 改动原因

本轮目标是把“会随业务变化的用户可见文案”前移到服务端契约：

- 客户端只保留模块标题、固定导航、固定类型 key。
- 首页提醒、INR 状态、检测方式说明、校正前后展示说明、完成服药后的明日剂量确认等业务文案由服务端返回。
- API 契约中明确 `displayText` / `copy` / `locale`，为后续多语言做准备。

## 3. 改动内容

### 3.1 服务端模型与服务

- `HomeSummary` 增加首页展示文案块。
- `INRRecord` 增加记录级展示文案。
- INR 列表响应增加趋势、记录标签、目标范围等展示文案。
- 服务层新增文案组装 helper，默认返回中文 `zh-CN`。
- 服务端测试覆盖首页、INR 创建、INR 列表中的文案字段。

### 3.2 OpenAPI 契约

- `UserSettings` 增加 `displayText` 字段，服务端返回 `SettingsDisplayText`（OpenAPI schema 名称）用于设置页检测方式、偏移量、检测周期等文案。
- 对首页、INR 记录和 INR 列表响应声明服务端展示文案字段。
- 保持 INR 校正值主显示、原始值弱显示的字段语义。

### 3.3 静态客户端原型

- `app.js` 新增模拟服务端 payload。
- 首页“最新 INR”“下次检测”“超明显提醒”“完成服药后选择明日剂量”等文案从 payload 读取。
- INR 页面异常分层、趋势图图例、记录说明从 payload 读取。
- 设置页检测方式、偏移量、检测周期、完成后规则等说明从 payload 读取。

### 3.4 文档

- UI 文档补充“文案来源”和“多语言准备”验收边界。
- 技术方案补充服务端文案契约原则。
- 文档站首页新增本报告入口。

## 4. 改动结果与效果

- 业务文案有了清晰归属：服务端负责业务语义，客户端负责布局与固定导航。
- 后续真实 Flutter/小程序接入时，可直接消费 `displayText` / `copy`，避免三端重复维护业务中文。
- 保持此前 11 条产品需求不回退：最近 INR、下次检测、明日剂量选择、异常分层、检测方式、偏移量、检测周期、强提醒、双曲线趋势仍保留。

## 5. 验证清单

本报告随代码一起提交，最终以实际命令输出为准：

- 服务端：`go test ./... && go vet ./...`。

- 小程序：`cd miniapp && npm test`。
- 安全扫描：新增 diff 中检查敏感词、危险函数和 whitespace。
- 独立代码评审：通过子代理复核后再提交部署。
- Cloudflare：部署后访问线上首页、报告页、微信首页和 INR 页面 smoke。

## 6. 后续建议

1. 将服务端文案 helper 演进为可配置语言包。
2. OpenAPI 生成 TS/Dart 类型，端侧禁止散落业务文案。
3. 增加 `Accept-Language` 或用户设置语言字段。
4. 对高风险医疗提示建立集中审核流程，避免端侧私自改文案。
